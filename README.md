# S-GLOBAL DOMINION v200.10

## Обзор системы

S-GLOBAL DOMINION - это SaaS-платформа для управления автопарком, финансами и аналитикой. Система построена на архитектуре FastAPI + PostgreSQL + Redis и оптимизирована для высоких нагрузок.

## Требования

- Docker
- Docker Compose
- 2 ГБ RAM минимум
- 10 ГБ свободного места на диске

## Быстрый старт

1. Клонируйте репозиторий:
```bash
git clone https://git.s-global.space/dominion.git
cd dominion
```

2. Запустите систему с помощью скрипта:
```bash
./start_dominion.sh
```

3. Откройте браузер и перейдите по адресу:
```
http://localhost:8001
```

4. Войдите в систему с учетными данными:
```
Логин: master
Пароль: MasterSpartak777!
```

## Архитектура системы

S-GLOBAL DOMINION использует трехуровневую архитектуру:

1. **Уровень представления**: HTML/CSS/JS с использованием D3.js для визуализации
2. **Уровень приложения**: FastAPI (асинхронный Python)
3. **Уровень данных**: PostgreSQL + Redis

## Особенности системы

- **Составные ключи**: Все запросы к БД используют составной ключ (park_name, yandex_car_id)
- **Асинхронность**: Все операции с БД выполняются асинхронно
- **Безопасность**: JWT-аутентификация для защиты API
- **Масштабируемость**: Docker-контейнеры с возможностью горизонтального масштабирования

## Резервное копирование

Система поддерживает автоматическое восстановление из резервной копии:

```bash
# Восстановление из резервной копии
./start_dominion.sh
# Выберите 'y' при запросе о восстановлении
```

## Устранение неполадок

### Проблема: Ошибка 500 при входе
Решение: Проверьте, что база данных инициализирована:
```bash
docker-compose exec app python -m app.create_db
docker-compose exec app python -m app.create_master_user
```

### Проблема: Перекрытие панелей в интерфейсе
Решение: Очистите кэш браузера или используйте режим инкогнито.
